# بازیابی اضطراری — اگه Hugging Face Space بن/محدود شد

هدف: از بن‌شدن تا برگشت کامل پایپ‌لاین، چیزی در حد چند دقیقه، نه بازسازی از صفر.
چون همه‌چیز (Dockerfile، entrypoint، و ۶ فایل JSON ورک‌فلوهای n8n) از قبل در گیت هست.

## پیش‌نیاز (همین الان آماده است)

- ✅ `infra/huggingface-n8n/Dockerfile` — قابل اجرا روی هر هاست Docker
- ✅ `infra/huggingface-n8n/entrypoint.sh` — کلون ریپو + materialise .env + start n8n
- ✅ `infra/huggingface-n8n/n8n-workflows/*.json` — هر ۶ ورک‌فلو
- ✅ Supabase (دیتابیس n8n) — مستقل از HF است، بن‌شدن Space بهش آسیبی نمی‌زنه

## مرحله به مرحله (گزینه‌ی پشتیبان: Render.com)

> ⚠️ مستندات Render می‌گه «بدون کارت»، ولی چند کاربر واقعی گزارش دادن
> موقع ساخت Web Service ازشون کارت خواسته شده. **اول امتحان کنید** —
> اگه کارت خواست، فوراً به من خبر بدید گزینه‌ی بعدی رو بررسی کنیم
> (مثلاً یک هاست Docker رایگان دیگه که در اون لحظه بدون کارت باشه).

1. حساب Render بسازید (با GitHub sign-in، بدون وارد کردن چیز دیگه‌ای).
2. **New → Web Service** → ریپوی torshield-ir رو وصل کنید.
3. Render خودش `infra/huggingface-n8n/Dockerfile` رو تشخیص می‌ده —
   اگه نداد، مسیر Dockerfile رو دستی به همون آدرس ست کنید.
4. Environment → همون متغیرهایی که در HF Space Secrets داشتید رو اینجا
   هم وارد کنید (GH_PAT, GH_OWNER, GH_REPO, DB_POSTGRESDB_*, و همه‌ی
   متغیرهای `configs/env_template.sh`) — همون اسم‌ها، همون مقادیر.
5. Deploy. وقتی بالا اومد، آدرس جدید n8n رو باز کنید — چون به همون
   Supabase وصله، **تمام ورک‌فلوها و credential ها از قبل آماده‌ان** —
   چیزی از نو ساخته نمی‌شه.
6. در GitHub → Settings → Webhooks، فقط آدرس Webhook رو از HF به آدرس
   جدید Render عوض کنید (یک فیلد، نه بازسازی).
7. اگه به هر دلیلی Supabase هم در دسترس نبود (بعید، ولی برای اطمینان
   کامل): ورک‌فلوها رو از همین فایل‌های JSON در `n8n-workflows/`
   دوباره Import کنید — طبق `n8n-workflows/README.md`.

## همین الان لازم نیست انجامش بدید

این یک نقشه‌ی آماده است، نه یک کار فوری. فقط بدونید کجاست. اگه روزی
HF بن شد، همین فایل رو باز کنید و مرحله‌به‌مرحله جلو برید.
