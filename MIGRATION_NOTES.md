# Python-to-Rust Migration Notes

## Phase 0 audit baseline

This repository is not ready for Python-file deletion yet. No Python behavior has
been removed in this change. The first enforceable migration artifact is the
static Phase 0 inventory generated by `tools/migration_audit.py` at
`data/migration_phase0_inventory.json`.

The audit records every Python file, top-level function/class/method contract
available from the AST, import edges, declared third-party requirements, and the
pytest-marker to Cargo-feature mapping required for later parity tests.

## Current deletion status

No `.py` file has been deleted. Future deletions must be gated on a parity test
that invokes the Python original and Rust replacement on the same branch-covering
golden inputs, including error paths.
