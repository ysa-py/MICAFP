## 5. Rollback Procedures

### 5.1 Provider Failure

If a provider fails:
1. The circuit breaker opens automatically after 5 consecutive failures
2. The gateway waterfall automatically skips open circuits
3. If all providers fail, LocalAIEngine provides degraded fallback
4. To manually reset a circuit breaker, restart the application

### 5.2 Configuration Rollback

1. Revert the environment variable to the previous value
2. Restart the application or re-run the workflow
3. Verify health check passes after rollback

## 6. Common Issues and Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| HTTP 401 from Portkey | Invalid API key | Check key format (pk- prefix), verify in Portkey dashboard |
| HTTP 404 from Cerebras | Invalid model name | Model discovery auto-fixes; check CEREBRAS_MODELS fallback list |
| HTTP 400 from CF AI Gateway | Invalid URL or model | Verify CF_AI_GATEWAY_URL includes account_id; model selector handles this |
| HTTP 403 with error 1010 | Cloudflare bot protection | User-Agent header is auto-set; if persists, check IP reputation |
| All providers failing | Network or credential issue | Check network connectivity, verify all API keys, run health check with --verbose |
| Circuit breaker stuck open | Persistent provider failure | Check provider status externally; circuit auto-recovers after 5 minutes |
| LocalAIEngine used often | All external providers down | Verify API keys and provider availability; LocalAIEngine is degraded mode |
